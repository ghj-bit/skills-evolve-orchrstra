# 高教社杯2020B 数学建模问题分析

> **分析目标**：为后续求解 agent 提供完整的题目理解基础，使其脱离原始题面即可开始工作。

---

## 1. 题目背景与研究对象

本题来源于一个策略游戏场景：玩家穿越沙漠从起点到达终点。玩家需要在负重限制下，通过购买水和食物维持生存，在矿山挖矿赚取资金，在村庄补充高价资源，最终目标是最大化剩余资金。游戏的核心矛盾在于**资源管理与路径规划之间的权衡**——携带更多资源增加生存保障但减少负重余量，绕道矿山/村庄可能获得资金/补给但增加时间与消耗。

游戏设定以"天"为基本时间单位，每天受"晴朗""高温""沙暴"三种随机天气之一影响（不同天气消耗不同），沙暴日强制原地停留。本题逐步升级：从单玩家全知天气（问题1），到单玩家仅知当日天气（问题2），再到多玩家协作与竞争场景（问题3），形成三个层次六个关卡的递进结构。

研究该问题的价值在于：其数学模型可推广至不确定环境下的资源调度、路径优化和多人博弈等现实问题。

## 2. 原始问题逐条拆解

### 问题一：单玩家已知全部天气的最优策略

- **原文表述**："假设只有一名玩家，在整个游戏时段内每天天气状况事先全部已知，试给出一般情况下玩家的最优策略。求解附件中的'第一关'和'第二关'，并将相应结果分别填入 Result.xlsx。"
- **问题类型**：优化（路径规划 + 资源分配 + 资金最大化）
- **核心目标**：在天气完全已知条件下，给出通用最优策略框架，并针对第一关和第二关给出数值解
- **输入依赖**：
  - 题目规则（第1-8条）
  - 第一关参数与地图（level_1_info.md）
  - 第一关天气序列（level_1_weather.csv，30天）
  - 第二关参数与地图（level_2_info.md）
  - 第二关天气序列（level_2_weather.csv，30天）
  - Result.xlsx 模板
- **输出要求**：将第一关和第二关的结果填入 Result.xlsx（含每日所在区域、剩余资金数、剩余水量、剩余食物量）
- **关键约束**：
  - 天气事先全部已知（确定性优化问题）
  - 必须给出"一般情况下"的通解，而非仅是两关的特解
  - 第0天在起点可购买一次（基准价格），不能在起点多次购买
  - 规则(6)："玩家可在起点停留或回到起点，但不能多次在起点购买资源"——意味着起点不能像村庄一样随时补给

### 问题二：单玩家仅知当日天气的最佳策略

- **原文表述**："假设只有一名玩家，玩家仅知道当天的天气状况，可据此决定当天的行动方案，试给出一般情况下玩家的最佳策略，并对附件中的'第三关'和'第四关'进行具体讨论。"
- **问题类型**：序贯决策 / 随机环境下的优化（无未来天气信息）
- **核心目标**：在仅知当日天气的不完全信息下，给出通用最佳策略框架，并对第三关和第四关进行具体讨论
- **输入依赖**：
  - 题目规则（同问题一）
  - 第三关参数与地图（level_3_info.md）
  - 第三关天气信息：仅知当日天气，已知**10天内不出现沙暴**（无完整天气序列）
  - 第四关参数与地图（level_4_info.md）
  - 第四关天气信息：仅知当日天气，已知**30天内较少出现沙暴**（无完整天气序列）
- **输出要求**：对第三关和第四关进行具体讨论（题目未要求填入Result.xlsx，但要求给出策略分析）
- **关键约束**：
  - 玩家仅知当日天气（后验信息，非预测信息）
  - "具体讨论"与问题一的"求解"措辞不同——输出形式可能更偏策略描述而非精确数值
  - 第三关时间仅10天且无沙暴，第四关30天且沙暴较少——时间压力和不确定性水平显著不同

### 问题三(1)：多玩家全知天气、计划锁定

- **原文表述**："假设在整个游戏时段内每天天气状况事先全部已知，每名玩家的行动方案需在第 0 天确定且此后不能更改。试给出一般情况下玩家应采取的策略，并对附件中的'第五关'进行具体讨论。"
- **问题类型**：多主体优化（合作或非合作多玩家决策）
- **核心目标**：在天气已知 + 行动方案锁定条件下，给出多玩家通用策略，并针对第五关（2人）具体讨论
- **输入依赖**：
  - 题目规则 + 多玩家规则（第3段）
  - 第五关参数与地图（level_5_info.md，与第三关同一地图）
  - 第五关天气序列（level_5_weather.csv，10天）
  - 玩家数 n=2
- **输出要求**：对第五关进行具体讨论
- **关键约束**：
  - 每名玩家的行动方案在第0天确定后不可更改（开环控制，无反馈）
  - 多玩家规则适用：
    - k人同天从A走到B：每人消耗 = 基础消耗 × 2k
    - k人同天在同一矿山挖矿：每人消耗 = 基础消耗 × 3，每人收益 = 基础收益 / k
    - k人同天在同一村庄购买：每箱价格 = 基准价格 × 4
  - 上述多玩家惩罚仅在 k≥2 时触发

### 问题三(2)：多玩家仅知当日天气、逐日信息共享

- **原文表述**："假设所有玩家仅知道当天的天气状况，从第1天起，每名玩家在当天行动结束后均知道其余玩家当天的行动方案和剩余的资源数量，随后确定各自第二天的行动方案。试给出一般情况下玩家应采取的策略，并对附件中的'第六关'进行具体讨论。"
- **问题类型**：多主体序贯决策 / 不完全信息动态博弈（信息共享）
- **核心目标**：在仅知当日天气 + 每日结束后共享信息的条件下，给出多玩家通用策略，并针对第六关（3人）具体讨论
- **输入依赖**：
  - 题目规则 + 多玩家规则
  - 第六关参数与地图（level_6_info.md，与第四关同一地图）
  - 第六关天气信息：仅知当日天气，已知**30天内较少出现沙暴**（无完整天气序列）
  - 玩家数 n=3
- **输出要求**：对第六关进行具体讨论
- **关键约束**：
  - 每日行动后所有玩家信息透明（知道他人行动和剩余资源，但不知道他人的策略/目标函数）
  - 多玩家规则适用（同三(1)）
  - "信息共享"改变了决策的信息结构——可从他人行为推断

| 序号 | 规则 | 来源 |
|------|------|------|
| 1 | 时间单位：天。第0天位于起点，截止日期或之前到达终点 | 规则(1) |
| 2 | 资源：水和食物，最小计量单位：箱。每天水+食物质量 ≤ 负重上限。未到终点水或食物耗尽 = 游戏失败 | 规则(2) |
| 3 | 天气三种：晴朗、高温、沙暴。全沙漠天气相同 | 规则(3) |

## 5. 关键概念与术语词典

- **天（day）**：基本时间单位。第0天为起始，玩家位于起点。截止日期当日及之前到达终点均为有效。
- **负重上限**：玩家每天可携带的水和食物的**质量之和**的最大值。所有关均为1200千克。注意是质量限制而非箱数限制。
- **基础消耗量**：玩家在某天气下**原地停留一天**消耗的一种资源的箱数。水与食物各有自己的基础消耗量。行走消耗 = 基础消耗 × 2；挖矿消耗 = 基础消耗 × 3（仅水+食物均3倍，亦或是挖矿本身仅翻倍一次？题目原文："如果挖矿，消耗的资源数量为基础消耗量的3倍"——指水和食物各为3倍）。
- **基础收益**：在矿山挖矿一天获得的资金量。各关不同：第一/二/四/六关1000元/天，第三/五关200元/天。
- **基准价格**：在起点（仅第0天一次）以该价格购买资源的价格。水5元/箱，食物10元/箱。退回价为基准价格的一半。
- **起点购买规则 vs 村庄购买规则**：
  - 起点：仅第0天可用初始资金以基准价格购买一次。可返回起点但不能再次购买。
  - 村庄：可用剩余资金（包括挖矿所得）随时以 2×基准价格购买，无次数限制。
- **矿山到达日 vs 挖矿日**：到达矿山当天不能挖矿（规则7）——第一天如果在矿山，则那天只能在矿山停留（不挖矿或挖矿的规则不适用？准确说法是"到达矿山当天不能挖矿"，意味着若当天到达矿山只能停留不挖矿，消耗 = 基础消耗量）。
- **沙暴日挖矿**：规则(7)明确"沙暴日也可挖矿"——即沙暴日虽不能移动但可以在矿山进行挖矿活动。
- **晴朗 / 高温 / 沙暴**：三种天气类型。沙暴日：全体必须原地停留（不能移动）。所有区域天气相同。
- **多玩家关键概念**：
  - **同行惩罚**（k人同行A→B）：每人消耗 = 基础消耗 × 2k。注意：与单人行走的"2倍"对比，多人时从"2倍"变为"2k倍"。
  - **同矿惩罚**（k人同矿挖矿）：每人收益 = 基础收益 / k，每人消耗维持基础消耗 × 3。
  - **同村惩罚**（k人同村购买）：每箱价格 = 基准价格 × 4。
  - 上述惩罚均在 k≥2 时生效。k=1 时视为单人规则。
- **相邻 vs 有公共顶点**：有共享**边**界 = 相邻（可移动通过）。仅有共享**顶点**而无共享边 = 不相邻（不可移动通过）。
- **"最优策略" vs "最佳策略"**：题目原文问题1用"最优策略"，问题2用"最佳策略"，措辞差异可能暗示问题2因天气不确定无法求严格最优，而求某种准则下的最佳。
- **"求解" vs "具体讨论"**：问题1要求"求解"并将结果填入Excel；问题2和问题3要求"具体讨论"，未要求填表。输出粒度可能不同。
- **剩余资金数/水量/食物量**：指当日所需资源全部消耗完毕后，若有购买则指完成购买之后的数值（注2）。即每日结束时的余额。

## 8. 核心难点与风险点

### 8.1 理解难点

- **难点1：每日行动与消耗的时序问题。** 题目注2定义了"剩余"的计算口径，但每日内部"移动-消耗-购买-挖矿"这些行动的时间顺序并未完全明确。例如：玩家到达村庄当天，是在消耗后购买还是先购买后消耗？这直接影响负重约束和资金约束的计算。│ 可能导致不同建模者对同日消耗与购买的计算顺序不一致 │ 涉及所有子问题

- **难点2：第0天的天气和消耗。** 题目说"第0天玩家位于起点"且天气序列从第1天开始，但第0天是否有天气？是否有资源消耗？如果没有消耗，第0天仅做初始购买，则天气序列和消耗表从第1天开始。│ 若不明确，第0天是否消耗水/食物会影响初始购买量的计算 │ 涉及所有子问题，尤其是问题1的精确求解

- **难点3：问题3中玩家目标是合作还是非合作。** 题目用"玩家应采取的策略"这一中性表述，未指定是最大化总剩余资金（合作）还是各自最大化（非合作/博弈）。问题3(1)计划锁定+天气全知的设定更适合合作优化（类似集中式规划）；问题3(2)信息共享+逐日决策的设定更符合博弈论框架。│ 目标函数的不确定直接影响建模方向 │ 涉及问题3(1)和3(2)

- **难点4："一般情况下的最优/最佳策略"的范围界定。** 问题1要求"一般情况下玩家的最优策略"，"一般情况下"的边界在哪？是否需要对任意的地图拓扑、参数设定、天气序列都给出策略框架？还是仅在给定关卡参数范围内的"一般"？│ 策略的通用性程度决定了建模的抽象层次 │ 涉及所有子问题的"一般情况"部分

- **难点5：第三/四/六关天气仅有定性描述。** 第三关"10天内不出现沙暴"、第四/六关"30天内较少出现沙暴"——"较少"是模糊的定性词。问题2和问题3(2)要求"最佳策略"和"具体讨论"而非精确求解，似乎暗示不需要精确的天气概率模型，但策略仍需要建立在某种天气不确定性刻画之上。│ 天气不确定性的建模方式将显著影响策略结果 │ 涉及问题2和问题3(2)

### 8.2 求解风险

- **风险1（严重）：多玩家"同行惩罚"3×2k的误解。** 规则M2：k人同行A→B，每人消耗 = 基础消耗 × 2k。对比单人行走 = 基础消耗 × 2。容易误读为"多一个人的惩罚就是多2倍基础消耗"。实际：2人同行 = 基础消耗 × 4；3人 = 基础消耗 × 6。需要正确理解数学关系。│ 可能导致消耗计算错误，影响路径选择和策略判断 │ 涉及问题3(1)和3(2)

- **风险2（严重）：矿山"到达当天不能挖矿"与"沙暴日可挖矿"的组合。** 如果玩家在沙暴日到达矿山，该日因是到达日不能挖矿，且因是沙暴日不能移动——那么该日只能保持原地不挖矿，消耗=基础消耗。另一种情况：如果玩家已经在矿山（非到达日），遇到沙暴日，虽不能移动但可挖矿，消耗=基础消耗×3。│ 两种情况的消耗不同(1× vs 3×基础消耗)，资金收入也不同(0 vs 基础收益) │ 涉及所有子问题（特别是沙暴日较多的第一/二关）

- **风险3（中）：终点退回的"半价回收"影响策略。** 到达终点后剩余资源可按半价退回，这意味着多买资源存在"贬值风险"——每多买1箱水（5元买的），如果没用掉只能退2.5元。但也意味着不应该在终点前把资源恰好用完（退回仍有价值）。│ 可能影响最终资金最大化的最优资源余量 │ 涉及所有子问题

- **风险4（中）：村庄购买"2倍价格"与"负重上限"的联动约束。** 在村庄买资源虽然贵，但可以减少前期携带量以节约行走消耗（因行走消耗是基础消耗的2倍，多带资源意味着更多负重=可能需要更多天行走）。这构成了"多买多耗vs少带多买"的权衡。│ 不合理的简化可能忽略这一关键权衡 │ 涉及所有有村庄的关卡（第一、二、四、六关）

- **风险5（中）：不同天气下水和食物消耗的"不对称性"。** 以第一关为例：晴朗天水5食物7（总12箱，39kg）；高温天水8食物6（总14箱，36kg）；沙暴天水10食物10（总20箱，50kg）。水和食物的消耗比例在不同天气下不同，不能简单地以单一"总消耗"为单位进行计算——需要分别跟踪水和食物的箱数，因为两者的质量和价格也不同。│ 可能简化过度导致资源规划不满足实际约束 │ 涉及所有子问题

- **风险6（低）：第一关与第二关天气相同但不是同一个问题。** 第一关和第二关使用完全相同的30天天气序列，但地图完全不同（27节点不规则 vs 64节点六边形）。适合对比分析地图结构对策略的影响。若两个地图独立建模但天气共享，可考虑策略框架的通用性验证。│ 不构成直接风险，但值得关注 │ 涉及问题1

- **风险7（低）：问题2和问题3(2)的"讨论"措辞差异。** 问题1明确要求"求解"并填入Excel；问题2和3要求"具体讨论"。这可能意味着后者不需要精确数值解，但也不排除可以给数值解。│ 建模时不确定精度的预期水平 │ 涉及问题2和问题3

## 9. 可复用信息摘要

### 9.1 一句话概括

穿越沙漠的资源管理与路径规划问题：在负重、时间、天气、购买/挖矿机会的多重约束下，最大化单人或多人到达终点后的剩余资金。

### 9.2 核心要素速查

- **研究对象**：玩家（1名或多名）在沙漠地图上从起点到终点的每日行动决策
- **时间范围**：第0天至第30天（第一、二、四、六关）；第0天至第10天（第三、五关）
- **空间范围**：四种不同的地图（27节点、64节点六边形、13节点、25节点5×5网格），每个节点有类型（起点/终点/矿山/村庄/普通）
- **子问题数量**：3个主问题，其中问题3分为(1)和(2)，共4个独立要求；涉及6个关卡
- **主要输出**：
  - 问题1：通用策略 + Result.xlsx（第一关和第二关的每日数据）
  - 问题2：通用策略 + 第三关和第四关的讨论
  - 问题3(1)：通用策略 + 第五关讨论
  - 问题3(2)：通用策略 + 第六关讨论

### 9.3 关键数字与参数

- **负重上限**：1200 kg（所有关统一）
- **初始资金**：10000元（所有关统一）
- **基础收益（挖矿日收入）**：1000元/天（第一/二/四/六关）；200元/天（第三/五关）
- **水参数**（所有关一致）：3 kg/箱，基准价格5元/箱，退回价2.5元/箱
- **食物参数**（所有关一致）：2 kg/箱，基准价格10元/箱，退回价5元/箱
- **消耗倍数**：停留=1×，行走=2×，挖矿=3×
- **村庄溢价**：单人=2×基准价格，多人(k≥2)=4×基准价格
- **多人同行惩罚**：k人同路→每人消耗=基础消耗×2k
- **多人同矿惩罚**：k人同矿→每人收益=基础收益/k，每人消耗维持基础消耗×3
- **第一/二关资源基础消耗（箱/天）**：水(晴5,高8,沙10)；食物(晴7,高6,沙10)
- **第三/四/五/六关资源基础消耗（箱/天）**：水(晴3,高9,沙10)；食物(晴4,高9,沙10)
- **第一/二关天气统计（30天）**：晴9天，高温15天，沙暴6天
- **第五关天气统计（10天）**：晴6天，高温4天，沙暴0天

### 9.4 必须注意的事项

1. **资源是两维的不是一维的**：水和食物不能互相替代，消耗速率不同、价格不同、质量不同，必须分别跟踪箱数。负载约束是总重量（3×水箱数+2×食物箱数≤1200）。
2. **时序规则要精确定义**：第0天是否有消耗？每日内部"移动→消耗→购买→挖矿"的顺序如何？需要基于注2的说明确定一致的计算口径。
3. **多玩家的目标函数可能不是唯一的**：问题3未明确合作还是非合作。需要确定优化目标（总资金最大化、各自最大化、还是某种均衡策略）。
4. **"一般情况"需要可操作的抽象层次**：既要足够通用以覆盖同类型任意参数，又要足够具体以产生有价值的结论。可能需要在参数化表达和关卡特化之间找到平衡。
5. **矿山"到达日不挖矿"意味着最优策略可能包含"提前一天到达矿山，次日开始挖矿"的时序优化**，特别是在截止日期紧张时，到达日和挖矿日的安排直接影响收益。
